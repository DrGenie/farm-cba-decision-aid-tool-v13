What this tool does

This decision aid helps you compare a control treatment with alternative treatments using cost-benefit analysis and simple charts. It is designed for replicated field trials and demonstration sites where you have multiple plots or replicates for each treatment.

The tool turns your trial dataset into a small set of summary indicators that can be used in extension discussions, project reporting, and teaching.

Getting started

  1. Download the data template. Use the buttons in the Data & settings tab to download a blank template (TSV or Excel) with the correct column names, including a column that flags which rows are the control.
  2. Fill the template offline. Add one row per plot or replicate. Mark the control rows, enter the main cost and outcome fields, and leave optional columns blank if you do not use them.
  3. Upload your file. Use the upload box to select your completed template. The tool checks that key columns are present and that at least one control and one treatment are available.
  4. Review validation messages. Any problems (for example, missing required columns, no control, or no treatment rows) are listed clearly so you can fix the file. Optional variables can be left blank.
  5. View results and export. Once the data are accepted, use the Results and Charts tabs to explore the analysis, then export a Word-ready summary from the Exports & AI brief tab.

Better structured data gives better analysis, but the tool can still provide indicative results when some optional fields are missing. The focus is on keeping users in the tool even when their datasets are imperfect.

Minimum data required

The tool enforces only the minimum needed to run a basic analysis:

  • At least one row flagged as control.
  • At least one row flagged as a treatment.
  • Core cost and outcome fields used in the calculations populated for some rows.

Optional columns (for example rainfall, biomass, or other covariates) may be left empty. You do not need to delete unused columns. Calculations proceed using the information that is available in the core fields.

How the tool handles replicates

Most trials have multiple plots or replicates for each treatment. The tool:

  • Aggregates all rows for each treatment into an average per hectare.
  • Aggregates all control rows into a single control average.
  • Compares each treatment average against the control average.

This is noted again in the Results tab, so users understand that each table row represents an average over all plots, not a single plot.

How data quality is handled

Uploading data is often where users encounter friction. The tool is designed to keep users engaged even when their datasets are not perfect.

  • Clear messages explain any problems with the file (for example, missing required columns, no control identified, or no treatment rows).
  • The template can contain missing values in optional columns; the tool still runs and labels outputs as indicative when full cost-benefit analysis is not possible.
  • When only partial information is available, the tool falls back to a basic analysis of treatment means and differences relative to the control.

The aim is to convert data issues into simple steps the user can fix, rather than reasons to abandon the tool.

1. Download template

Start here if you do not already have a dataset in the expected format. The template includes all required columns plus optional fields for additional trial information.

You can open the template in Excel or another spreadsheet program, fill it offline, and then upload it here. Do not rename or delete the column headers.

2. Upload data

Upload a tab-separated (.tsv) or comma-separated (.csv) file that follows the template.

3. Template columns

The template uses one row per plot or replicate. Each column has a specific role in the analysis. Hover over a column name to see what to enter.

Optional columns may be left blank. The tool will still run using the core fields.

4. Scenario settings

These settings apply to all treatments in the uploaded dataset. They can be edited without re-uploading the data.

Indicative results
